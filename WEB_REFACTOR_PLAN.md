# ytdl-web 改造文档（先文档后开发）

## 1. 改造目标
- 从 tkinter 单文件桌面程序，升级为 Flask 分层 Web 应用。
- 默认开启代理：`http://127.0.0.1:7890`。
- 前端分层：HTML / CSS3 / JavaScript 独立文件，整体视觉更酷炫。
- 界面国际化：简体中文、繁体中文、英文、日语、韩语、法语、西班牙语，可扩展。
- 配置持久化到本地 SQLite（代理、语言、下载路径、常用选项）。
- 基于 yt-dlp 已有能力扩展更多可视化选项与控制项。
- 支持 Docker 部署，便于本地和服务器统一运行。
- 支持服务端部署后“下载到浏览器本地”交付模式。
- 下载合集时支持全选、反选、按条目局部选择下载。

## 2. 架构分层设计

### 2.1 后端分层（Flask）
- `app.py`：应用入口与工厂初始化。
- `app/routes/`：路由层，仅做参数接收与响应输出。
- `app/services/`：业务层，处理任务调度、下载逻辑、信息提取。
- `app/core/`：核心能力（yt-dlp 适配、任务状态管理、i18n、工具函数）。
- `app/repositories/`：数据访问层（SQLite 配置读写）。
- `app/models/`：数据模型/DTO（配置对象、任务对象）。
- `app/templates/`：HTML 模板。
- `app/static/css`、`app/static/js`：样式与脚本资源。

### 2.2 前端分层
- `index.html`：结构层，负责布局与组件占位。
- `app.css`：视觉层，渐变、玻璃拟态、动画与响应式布局。
- `app.js`：行为层，API 调用、状态管理、轮询、i18n 文案切换。
- `i18n.js`：前端多语言文案资源与切换逻辑。
- `modal.js`：模态框与确认交互（可并入 `app.js`，但逻辑需独立模块化）。

## 3. 功能规划

### 3.1 默认代理策略
- 系统初始代理为 `http://127.0.0.1:7890`。
- 首次运行时写入 SQLite 默认配置。
- 前端可切换 `none/http/socks5`，并可修改主机与端口。

### 3.2 国际化（i18n）
- 支持语言：
  - 简体中文 `zh-CN`
  - 繁体中文 `zh-TW`
  - 英文 `en`
  - 日语 `ja`
  - 韩语 `ko`
  - 法语 `fr`
  - 西班牙语 `es`
- 语言切换后即时刷新页面文案，不刷新页面。
- 用户语言偏好保存到 SQLite。

### 3.3 yt-dlp 能力扩展（首批）
- 信息获取：视频/合集基础信息、可用格式、字幕、音轨。
- 下载选项：
  - 视频格式选择与音频轨道选择
  - 字幕下载（人工/自动）
  - 音频提取格式（mp3/aac/flac）
  - 输出模板（默认 `%(title)s.%(ext)s`）
  - 元数据写入（write metadata）
  - 缩略图下载（write thumbnail）
  - 限速（rate limit）
  - 重试次数（retries）
  - 并发分片数（concurrent fragment downloads）
  - 合集条目选择下载（playlist_items）
  - 文件下载链接回传（便于浏览器拉取到本地）

### 3.4 性能优化策略
- 服务端线程池执行下载任务，避免阻塞 Flask 请求线程。
- 任务状态与日志内存环形缓存，限制最大长度。
- 轮询接口返回增量友好结构，减少前端处理负担。
- 配置读取加缓存，减少 SQLite 高频查询。
- 前端轮询间隔动态调整（任务活跃时快、空闲时慢）。

### 3.5 配置持久化（SQLite）
- 表：`app_settings`
  - `id`
  - `language`
  - `download_path`
  - `proxy_type`
  - `proxy_host`
  - `proxy_port`
  - `updated_at`
- 表：`download_presets`（可选扩展）
  - 常用下载参数预设，支持快速套用。

### 3.6 本地部署/服务端部署统一下载体验
- 本地部署：可直接写入本机目录，也可点击文件链接保存到浏览器默认下载目录。
- 服务端部署：任务在服务端落盘后，前端通过下载链接把文件拉到用户本地。
- 任务结果中返回文件清单与下载 token，前端展示“下载到本地”按钮。
- 支持单文件直接下载与多文件逐条下载。

## 4. API 设计（v1）

### 4.1 页面与健康检查
- `GET /`：返回首页。
- `GET /api/health`：服务健康状态与依赖状态。

### 4.2 配置相关
- `GET /api/settings`：读取当前配置。
- `PUT /api/settings`：更新配置并持久化到 SQLite。

### 4.3 信息与下载
- `POST /api/info`：获取视频/合集信息与可选项。
- `POST /api/download`：创建下载任务，返回 `task_id`。
- `GET /api/tasks/<task_id>`：查询任务状态/进度/日志。
- `POST /api/tasks/<task_id>/cancel`：取消下载任务。
- `GET /api/tasks/<task_id>/files`：获取已完成文件列表。
- `GET /api/files/<file_token>`：下载服务端文件到浏览器本地。

## 5. 页面体验设计
- 风格：深色科技风 + 渐变光效 + 轻动效反馈。
- 区域：
  - 顶部导航（语言、主题标识）
  - 左侧菜单（下载/任务/设置）
  - 链接与信息区域
  - 下载参数区域
  - 任务状态与进度区域
  - 日志区域
- 交互：
  - 可视化状态标签（queued/running/success/error/cancelled）
  - 任务运行时按钮状态联动
  - 错误提示统一弹层/条幅
  - 设置入口独立菜单，使用模态框管理配置
  - 下载路径修改使用弹框确认，防误操作
  - 合集条目选择器模态框（全选/反选/局部勾选）
  - 完成后弹出“下载到本地”模态框

## 6. Docker 部署设计

### 6.1 Dockerfile
- 基础镜像：`python:3.12-slim`
- 安装依赖：Flask、yt-dlp
- 暴露端口：`8000`
- 启动命令：`python app.py`

### 6.2 docker-compose.yml
- 服务：`ytdl-web`
- 端口映射：`8000:8000`
- 卷挂载：下载目录、SQLite 数据文件目录
- 重启策略：`unless-stopped`
